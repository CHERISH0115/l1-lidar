# 摘要
随着自主移动平台在室内定位、智能巡检、服务机器人等场景的广泛应用，实时、稳定、低开销的同步定位与地图构建（SLAM）技术成为核心研究方向。传统激光SLAM系统常依赖外接惯性测量单元与重型可视化工具，存在硬件集成复杂、运行开销高、嵌入式部署困难等问题。为此，本文设计并实现一套基于宇树L1激光雷达与ROS2的激光-IMU紧耦合SLAM系统。系统以树莓派为核心计算平台，利用宇树L1雷达内置IMU完成数据融合，省去外接惯导模块，简化硬件结构；针对RViz可视化工具性能开销过大的问题，采用C++与Qt框架重构轻量化显示界面，提升系统运行效率；同时设计基于STM32与LoRa模块的远程遥控终端，实现系统启动、停止控制及状态信息OLED实时显示，增强系统实用性。

实验结果表明，本系统可在室内环境完成稳定建图与定位，自研Qt界面较RViz降低约40%以上CPU占用，遥控终端响应可靠、通信稳定，整体满足低成本、低功耗、轻量化的嵌入式SLAM应用需求。

**关键词**：激光SLAM；ROS2；宇树L1；激光-IMU融合；Qt可视化；STM32遥控

# ABSTRACT
With the wide application of autonomous mobile platforms in indoor positioning, intelligent inspection, and service robots, real-time, stable, and low-overhead Simultaneous Localization and Mapping (SLAM) technology has become a core research direction. Traditional laser SLAM systems often rely on external inertial measurement units and heavy visualization tools, resulting in problems such as complex hardware integration, high running overhead, and difficulty in embedded deployment. For these reasons, this paper designs and implements a laser-IMU tightly coupled SLAM system based on Unitree L1 LiDAR and ROS2.

The system uses Raspberry Pi as the core computing platform, and uses the built-in IMU of Unitree L1 LiDAR to complete data fusion without external IMU, simplifying the hardware structure. To solve the problem of excessive performance overhead of RViz, a lightweight display interface is reconstructed using C++ and Qt framework, improving system efficiency. A remote control terminal based on STM32 and LoRa module is designed to realize start/stop control and OLED real-time display of status information, enhancing the practicability of the system.

Experiments show that the system can complete stable mapping and positioning in indoor environment. The self-developed Qt interface reduces CPU occupancy by more than 40% compared with RViz. The remote control terminal responds reliably and communicates stably. The whole system meets the requirements of low-cost, low-power and lightweight embedded SLAM applications.

**Key words**: Laser SLAM; ROS2; Unitree L1; Laser-IMU Fusion; Qt Visualization; STM32 Remote Control

# 第一章 绪论

## 1.1 研究背景及意义
近年来，自主定位与环境感知技术快速发展，同步定位与地图构建（SLAM）作为移动机器人实现自主导航的核心技术，被广泛应用于室内巡检、仓储物流、服务机器人、智能小车等场景。在各类SLAM技术路线中，激光SLAM因具备精度高、鲁棒性强、不受光照影响等优势，成为室内结构化环境中的主流方案。

传统激光SLAM系统通常需要独立激光雷达与外接IMU进行数据融合，以提升运动状态估计的稳定性。但外接IMU会增加硬件接线复杂度、标定难度与系统成本。同时，主流SLAM系统依赖RViz等工具进行实时点云与轨迹显示，在树莓派等嵌入式平台上运行时，会出现CPU占用高、帧率低、界面卡顿等问题，严重影响系统实时性。此外，多数SLAM部署方案缺乏轻量化远程交互手段，难以实现便捷启停与状态监控。

基于上述问题，本文依托**宇树L1激光雷达（内置IMU）**、**ROS2**、**树莓派**、**STM32+LoRa**与**Qt**，设计一套结构简洁、低开销、可远程操控的激光-IMU紧耦合SLAM系统。本课题的研究意义主要体现在以下方面：
1. **硬件简化**：直接使用雷达内置IMU完成融合，无需额外惯导，降低成本与集成难度。
2. **性能提升**：自研Qt轻量化界面替代RViz，显著降低嵌入式平台资源占用。
3. **工程实用**：增加STM32+LoRa远程遥控与OLED状态显示，提升系统易用性与可部署性。
4. **教学与落地**：符合本科毕设工程实践要求，可直接迁移至小车、巡检机、机器人等平台。

## 1.2 国内外研究现状
本节从**激光SLAM技术**、**激光-IMU融合定位**、**机器人可视化工具**、**嵌入式远程控制**四个方向展开综述。

### 1.2.1 激光SLAM技术研究现状
激光SLAM通过激光雷达获取环境点云，实现位姿估计与地图构建。早期经典算法以GMapping、Hector-SLAM、Cartographer为代表。GMapping基于粒子滤波，适合小型室内环境，但计算量大、大场景易退化；Hector-SLAM基于图优化匹配，无需IMU，但对雷达频率要求高；Cartographer引入子图匹配与闭环优化，建图精度高，但资源消耗较大。

随着激光雷达与计算平台发展，轻量化激光SLAM成为趋势。LOAM、LeGO-LOAM、Point-LIO等算法先后提出，通过特征点提取与紧耦合优化，大幅提升实时性与鲁棒性。Point-LIO针对高动态与振动场景优化，特别适合集成于机器人与小型移动平台，成为本文选用的核心算法。

### 1.2.2 激光-IMU融合定位研究现状
激光雷达在快速运动、遮挡、振动下易出现运动畸变与点云模糊，而IMU可提供高频运动预测，因此激光-IMU融合成为提升定位稳定性的主流方式。传统方案多采用“激光雷达+独立IMU”组合，需进行外参标定与时间同步。近年来，一体化雷达（内置IMU）逐渐普及，出厂已完成初步标定，可大幅降低开发难度。

国内外研究表明，紧耦合融合方式能更充分利用IMU高频输出，抑制振动与快速运动带来的误差。Point-LIO、LIO-SAM、FAST-LIO2等均采用紧耦合架构，在无人机、地面机器人上表现优异。

### 1.2.3 机器人可视化工具研究现状
机器人可视化是SLAM调试与展示的关键环节。ROS/RViz是目前最常用工具，功能全面但组件繁多，在树莓派等低算力平台上运行时帧率低、占用高。为解决该问题，国内外研究转向轻量化可视化：部分采用Python简化渲染，部分采用C+++OpenGL/Qt实现专用界面。专用界面可按需裁剪功能，显著提升帧率与流畅度，更适合嵌入式部署。

### 1.2.4 嵌入式远程控制研究现状
嵌入式SLAM系统常需要远程启停、状态监控等功能。传统方案多采用Wi-Fi，但穿墙能力弱、易受干扰。LoRa模块具备低功耗、远距离、抗干扰强等优势，适合无线遥控。STM32因其低成本、外设丰富，常被用作遥控终端主控，配合OLED可实现状态显示、按键控制、数据上传等功能，已广泛应用于机器人、物联网终端与智能小车系统。

## 2.2 系统整体方案设计
### 2.2.1 设计目标
1. 实现基于宇树L1雷达内置IMU的激光-IMU紧耦合SLAM。
2. 在树莓派+ROS2平台稳定运行Point-LIO建图算法。
3. 开发C++/Qt轻量化可视化界面，降低系统开销。
4. 实现STM32+LoRa远程启停控制与OLED状态显示。
5. 整体系统低成本、低功耗、轻量化、可部署。

### 2.2.2 硬件架构
系统硬件由四部分组成：
1. **感知层**：宇树L1激光雷达（内置IMU）。
2. **计算层**：树莓派（运行ROS2、Point-LIO、Qt界面）。
3. **遥控层**：STM32 + LoRa模块 + OLED显示屏 + 按键。
4. **电源层**：5V稳压供电模块。

硬件数据流向：
**宇树L1 → 串口/网口 → 树莓派 → 处理 → Qt界面显示**
**树莓派 ← LoRa ← STM32遥控终端**

### 2.2.3 软件架构
系统采用分层模块化设计：
1. **驱动层**：宇树L1雷达驱动、ROS2消息收发、串口驱动。
2. **SLAM层**：Point-LIO激光-IMU紧耦合建图。
3. **可视化层**：C++/Qt自研界面（点云、位姿、状态）。
4. **控制层**：LoRa通信协议解析、遥控指令执行、状态回传。
5. **交互层**：STM32按键、OLED显示、指令封装。

### 2.2.4 数据流程
1. 雷达输出点云+IMU数据。
2. 数据进入ROS2话题。
3. Point-LIO进行状态估计与地图构建。
4. 结果发送至Qt界面渲染。
5. STM32通过LoRa发送启停指令。
6. 系统状态回传至OLED显示。

## 2.3 宇树L1激光雷达与内置IMU特性分析
### 2.3.1 雷达测距原理
宇树L1采用激光飞行时间（ToF）测距，通过测量激光发射与接收的时间差计算距离，具有测距快、精度高、抗干扰强等特点。

### 2.3.2 内置IMU优势
雷达内置IMU可直接输出加速度、角速度，无需额外标定与同步，简化硬件结构，提升系统集成度。

### 2.3.3 动平衡与振动问题
宇树L1在高转速下存在一定机械振动，会导致点云轻微抖动与IMU噪声增大。本文通过软件滤波与算法鲁棒性缓解该影响。
